ТРЕНАЖЕР РЕЧЕНЬ — ЛОГІКА ТА СТРУКТУРА
1. ЗАГАЛЬНА ІДЕЯ

Тренажер генерує прості англійські речення на основі граматичних патернів.

Користувач:

бачить речення українською
збирає англійське речення зі слів (Easy / Easy+)
перевіряє правильність

Система:

не зберігає готові речення
генерує їх динамічно з наборів слів і патернів
2. СТРУКТУРА JSON ФАЙЛІВ

Всі дані лежать окремо, розбиті по типах.

2.1 subjects.json

Містить підмети з граматичною інформацією

Поля:

text — слово
person — 1 / 2 / 3
number — singular / plural / both
2.2 verbs.json

Містить дієслова з формами

Поля:

base — базова форма (work)
third — для 3 особи (works)
past — минулий час (worked)
ing — форма для continuous (working)
2.3 objects.json

Містить обʼєкти та обставини

Поля:

text — слово/фраза
type — place / time / object / misc
category — (наприклад: general / work / travel)
2.4 auxiliaries.json

Допоміжні слова

Поля:

text — do / does / did / is / are / will
tense — present / past / future
form — question / negative
person — масив (для узгодження)
number — масив
2.5 patterns.json

Каркаси речень

Поля:

tense_type — simple / continuous / perfect / perfect_continuous
tense_time — present / past / future
form — affirmative / negative / question
pattern — масив типу ["S","V","O"] або ["AUX","S","V","O"]
2.6 rules.json (опціонально)

Короткі правила

Поля:

tense_type
tense_time
form
text (коротке правило)

4. ЛОГІКА ГЕНЕРАЦІЇ РЕЧЕННЯ
КРОК 1. Вибір патерна

Фільтруються patterns:

по tense_type
по tense_time
по form

Випадково вибирається один патерн

КРОК 2. Вибір слів

Вибирається:

subject з subjects.json
verb з verbs.json
object з objects.json (з урахуванням category)
КРОК 3. Вибір auxiliary (якщо є)

Якщо патерн містить "AUX":

шукається відповідний auxiliary:
по tense
по form
по person і number підмета
КРОК 4. Формування речення

Проходження по pattern масиву:

S → subject.text
V → форма дієслова:

якщо Present Simple + 3 особа → third
якщо Past → past
якщо Continuous → ing
інакше → base

AUX → auxiliary.text
O → object.text

Все зʼєднується в рядок через пробіл

КРОК 5. Генерація українського речення

(або береться заготовлене, або простий переклад)
На MVP можна:

мати окремий ua-шаблон
або статичні варіанти
КРОК 6. Підготовка слів для UI

Береться правильне речення:

розбивається на слова
Easy:
тільки правильні слова
перемішані
Easy+:
додаються distractors (зайві слова)
типові помилки:
is / are
неправильна форма дієслова
КРОК 7. Відображення

Користувач:

бачить слова
клікає або перетягує
формує речення
КРОК 8. Перевірка

Порівнюється:

порядок слів
точний збіг з правильною відповіддю
КРОК 9. Фідбек
якщо правильно → показати правильний варіант
якщо ні → підсвітити помилку
