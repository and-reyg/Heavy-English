Окей, я перепишу все детально у **текстовому форматі**, щоб можна було вставити в `.txt`, включно з моїми рекомендаціями щодо структури файлів та поясненням категорій/ситуацій.

# Логіка та структура тренажера речень (текстовий формат для `.txt`)

## 1. Файли та структура даних

* Рекомендую **спочатку робити один великий JSON-файл на всі речення**, це зручно для MVP.
* Якщо база виросте сильно (тисячі речень), можна розбити **по часах / типах**:

```
sentences_present.json
sentences_past.json
sentences_future.json
```

* Потім у JS їх легко зчитати та об’єднати: просто завантажуєш всі файли у масив `sentences = [...present, ...past, ...future]`.
* Перевага одного файлу на старті: зручно перевіряти, редагувати та відразу фільтрувати.

---

## 2. Структура одного речення у JSON

```json
{
  "id": 1,                    // унікальний ідентифікатор
  "en": "I work at the office.", // англійське речення
  "ua": "Я працюю в офісі.",     // український переклад
  "tense_time": "present",        // час: present / past / future
  "tense_type": "simple",         // тип: simple / continuous / perfect / perfect_continuous
  "form": "affirmative",          // форма: affirmative / negative / question
  "subject": "I",                 // підмет, для аналітики/підказок
  "verb": "work",                 // головне дієслово, для підказок
  "category": "work",             // категорія: загальні теми (work, travel, school, home)
  "situation": "general"          // ситуація: більш детальний контекст, напр. робоча зустріч, аеропорт, навчання
}
```

### Пояснення полів `category` і `situation`

* **category** — широка група речень, за якою можна фільтрувати:

  * `general` — загальні повсякденні ситуації
  * `work` — робота, офіс
  * `travel` — подорожі, транспорт
  * `school` — навчання
  * `home` — будинок, сім’я

* **situation** — деталізація конкретної ситуації у межах категорії:

  * Для `work`: `meeting`, `email`, `office`
  * Для `travel`: `airport`, `hotel`, `train_station`
  * Для `school`: `class`, `homework`, `exam`
  * Для `general`: `shopping`, `restaurant`, `greetings`

* Мета: категорії потрібні для широкого сортування, ситуації — для більш тонкого вибору та розвитку тематики.

## 3. Логіка тренажера

1. **Завантаження даних**

   * JS завантажує JSON файл(и) і формує масив речень.
   * Фільтрує їх за вибраними користувачем опціями: час, тип, форма, категорія, ситуація.

2. **Вибір речення**

   * Випадкове або по порядку із відфільтрованого масиву.
   * Можна відстежувати `id`, щоб не повторювати одразу те саме речення.

3. **Відображення українського перекладу**

   * Користувач бачить тільки українське речення (`ua`) для перекладу.

4. **Складання англійського речення**

   * Користувач збирає речення через drag&drop або кнопки.
   * Для Easy+ можуть бути зайві слова для практики.

5. **Кнопка “Check”**

   * Перевіряє, чи збігається введене речення з `en` у JSON.
   * Підсвічує помилки (пропущене слово, зайве слово, неправильний порядок).

6. **Кнопка “Next”**

   * Завантажує наступне речення з відфільтрованого списку.

7. **Меню налаштувань**

```
- Час: Present / Past / Future / All
- Тип: Simple / Continuous / Perfect / Perfect Continuous
- Форма: Affirmative / Negative / Question
- Категорія: General / Work / Travel / School / Home
- Ситуація: (залежить від категорії)
- Режим: Easy / Easy+
- Показувати правило: on/off
- Показувати тип речення: on/off
- Показувати шкалу tense: on/off
```

---

## 4. Примітки щодо MVP

* **Почати з Present Simple**: 30–50 речень на форму (affirmative/negative/question)
* Далі додати Past Simple та Future Simple
* Потім Continuous / Perfect / Perfect Continuous
* Всі речення включають головні підмети (I, you, we, he, she) та 20–30 основних дієслів
* Додати категорії та ситуації для різноманітності

```txt
  { "id":2, "en":"He doesn't like coffee.", "ua":"Він не любить каву.", "tense_time":"present", "tense_type":"simple", "form":"negative", "subject":"He", "verb":"like", "category":"general", "situation":"food" },
  { "id":3, "en":"Do you play tennis?", "ua":"Ти граєш у теніс?", "tense_time":"present", "tense_type":"simple", "form":"question", "subject":"you", "verb":"play", "category":"general", "situation":"sports" }

```

* Пізніше можна розбити по файлах за часами, але об’єднати в JS масив дуже просто.
